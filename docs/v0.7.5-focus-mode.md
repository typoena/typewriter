# v0.7.5 — Focus mode (Pomodoro)

> Part of the [Typoena macro plan](macroplan.md). Requirements and targets:
> [qfd.md](qfd.md). Load-bearing decisions: [adr.md](adr.md).

**Status:** **DELIVERED 2026-07-17** — specced, built, and verified on the panel
the same day (5 focus tests, 245 editor / 29 keymap total; firmware **0.7.5**).
The silent timer, pause-gated break, +2 min grace cap (a continuous-typing block
force-broke at 27 s on device), word counting, and the `Ctrl-C`/`Ctrl-Q` rest
card all fired on hardware. An unplanned same-day feature, slotted after
[v0.7](v0.7-search-and-git.md) as a `.5` point release (the
[v0.2.5](v0.2.5-international-input.md) precedent).

A two-phase writing cycle: a silent **focus** block, then a full-screen **rest**
card that masks the text until you choose to continue. Distraction-free by
construction — there is **no visible countdown** at any point. The device tracks
the 25-minute block silently and imposes the break; the rest card appearing *is*
the alarm.

Surfaced as **"focus"** in the UI — the "Pomodoro Technique" name is a
registered trademark, so it stays in the docs only.

## The cycle

```
 OFF ──:focus──▶ FOCUS ──25 min, at next typing pause──▶ REST
                  │                                       │
             :focus (end)              Ctrl-C ─▶ next FOCUS block
                  │                    Ctrl-Q ─▶ OFF (end session)
                  ▼
                 OFF
```

- **FOCUS** — you edit normally; every mode still works. A monotonic timer runs,
  unseen. The side panel shows a small `focus` marker. Nothing repaints on a
  clock — the marker is static text.
- **Break due** — at 25 min the block is *due*. To avoid yanking the screen
  mid-word, the rest card drops at the **next typing pause** (or immediately if
  you are already idle). A +2 min grace cap forces it if you never pause.
- **REST** — a full-screen card masks the editor. Every key is swallowed except
  the two chords: **`Ctrl-C`** (start the next focus block) and **`Ctrl-Q`** (end
  the session, back to normal editing). Both exits are deliberate chords so a
  stray key can't end a break — a bare `c`/`q` and `Esc` do nothing behind the
  curtain, and the host drops the rest of the key batch on exit so a keyboard
  bump can't reach the buffer. Rest is **untimed** — you leave when you are
  ready. Masking the text is the point: it removes the temptation to keep working.

## The rest card

Full-screen, **themed**: a background fill with centred foreground text — a white
card in light theme, black in dark. Not a hard-coded black (a full black fill is
the harshest, slowest e-paper refresh with the most ghosting risk, for no gain).
Two lines:

```
                     Rest

              412 words · 26 min

        Ctrl-C continue  ·  Ctrl-Q quit
```

- **minutes** — block-end minus block-start via a monotonic `Instant`, always
  available offline (wall-clock time-of-day is not — it only exists after a
  Wi-Fi SNTP sync and is not plumbed to the UI thread).
- **words** — words added to the active buffer this block, `max(0, end − start)`.
  Best-effort: switching buffers mid-block resets the baseline, so the figure
  reflects the current buffer's contribution. It is a motivational number, not
  an audit.

Entering and leaving rest are one full refresh each (~1.9 s) — fine for a
deliberate break, not a hot path. `Ctrl-C` repaints the editor and starts a
fresh block (timer + word baseline reset); `Ctrl-Q` ends the session.

## Why no live timer

A ticking countdown is the wrong thing on e-paper: partials are ~630 ms, a
per-second repaint would flash and wear the panel, and it would need a new
idle-repaint tick in the main loop. The design sidesteps all of it — the elapsed
check is a **single one-shot threshold** in the existing idle/pause path, not a
repaint loop. You never see a clock; the break still gets imposed. This is the
point that reconciles "no live timer" with real timeboxing.

## Design notes

- **State split.** Rest is a new `Mode::Rest`, consistent with `Mode::Palette` —
  a full-screen modal already lives as a mode. Focus is an orthogonal
  `pomodoro_on` session flag, since you keep editing in Normal/Insert/Visual
  during a block. The pure editor core owns the mode transitions, the panel
  marker, the rest-card render, and key-swallowing — all host-testable. Timing
  (monotonic `Instant`) is the one host-side piece: `main.rs` holds the
  block-start instant + start word count, computes elapsed/words each loop, and
  calls `enter_rest(words, mins)` when a due block hits a pause (the same typing
  pause `refresh_stats` already keys off). `Ctrl-C` / `Ctrl-Q` in Rest emit
  `Effect::Focus*` so the host resets or clears its timer; on either exit the
  host drops the rest of the key batch. The two chords are new `keymap::Key`
  variants (`FocusContinue`/`FocusQuit`, HID `0x06`/`0x14` + Ctrl).
- **Rendering.** `draw_into` branches to a `draw_rest_card` path when
  `mode == Rest`; the panel marker is a new field in `draw_panel` gated on
  `pomodoro_on`.
- **Interactions.** Save, idle-save, and `:gp` / `:gl` are unaffected during
  FOCUS. A background sync that finishes during REST defers its snackbar until
  you continue (the panel is not visible). Nothing is persisted, so a power-pull
  mid-session just comes back OFF.

## Deferred (→ v1.x)

- Configurable focus length (25 default) — a duration pref cycled from the
  palette, on the `auto_sync` template.
- Timeboxed rest / auto-resume — v1 rest is untimed, manual `Ctrl-C`.
- Long break every 4th block.
- Cumulative writing stats — folds into the v1.x "writing stats" stretch item.
- A `> focus` palette entry — v1 is `:focus` only.
